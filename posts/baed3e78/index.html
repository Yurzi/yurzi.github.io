手工创建

为了能更加清晰的了解Maven项目的结构，对其进行手工配置。

创建目录结构

这是一般项目工程的结构，而java-project才将作为Maven工程结构的根目录。

  • src目录作为存放源的文件夹
    • mian：写程序
      • java：写java源程序
      • recourses：写配置文件或其他内容。
    • test：写测试程序
      • java
      • recourses：测试程序的配置文件和资源

手动创建这些目录。

在java中新建一个源程序，写入内容。写好相关测试内容。

创建pom.xml

在src的同目录下创建pom.xml，可以从其他项目或者其他地方复制一份。

标签名 含义
modelVersion maven项目对象的版本
groupId 全球唯一标识符
artifactId 生成的包的名称或项目名称
packing 打包/发布方式
version 项目版本
name 项目名称，用于生成文档
dependences 项目依赖

全内容pom.xml

构建项目

  1. 进入到pom.xml文件所在的目录
  2. 运行mvn compile第一次运行将下载对应的插件

测试项目

  1. 进入到pom.xml文件所在的目录
  2. 运行mvn test第一次运行将下载插件，同时注意作用域(scope)等问题

打包项目

  1. 进入到pom.xml文件所在目录
  2. 运行mvn package，第一次运行将下载插件

安装项目到仓库

  1. 进入到pom.xml文件所在目录
  2. 运行mvn install，第一次运行将下载插件
  3. 将根据定义的坐标，将包放到本地仓库
