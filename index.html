并发、进程、线程的基本概念

并发

两个或多个独立的活动同时进行

并发假象：单核CPU（上下文切换方式实现并发）

进程

计算机中的程序关于某个数据集合上的一次运行活动（exe的运行状态）

线程

每个进程都有一个主线程，并有且只有一个主线程

线程可以理解为代码运行的通道，路径，程序员可以创建多个线程

并发的实现

  1. 多进程间实现并发：主要解决进程间通信的问题
    1. 同一主机上：管道、文件、消息队列、内存共享
    2. 不同主机上：通过SOCKET网络通信实现
  2. 单个进程，多个线程实现并发，就是一个主线程多个子线程实现。一个进程中的所有线程共享部分内存空间，例如，全局变量、指针引用

线程创建的多种方式

C++线程创建的过程

本质上是在thread类的构造函数中传递函数指针。

  1. 在C++11后引用头文件#include<thread>若使用的是win32的Thread mode的编译器则无法调用thread类
  2. 创建线程：调用thread类去创建一个线程对象。对于该线程对象可以使用函数指针来进行初始化。即将一个函数的执行放入主线程去执行。注意点：创建一个线程，不做处理，会调用abort函数中止程序
  3. join()函数：加入，汇合线程，阻塞主线程，等待子线程执行结束才会回到主线程。每个线程只能join一次.
  4. detach()函数：分离 打破依赖关系。把子线程驻留后台。当线程detach后就无法join()
  5. joinable()函数：判断当前线程可否join或者detach，若可以则返回true。

创建线程的方式

  1. 普通函数的方式：如上
  2. 通过类和对象来创建线程：需要重载对象的operator()且不能传参。std::thread t1(obj);或者``std::thread t1((classname()));`classname()将会创建一个无名对象。
  3. 通过C++[Lambda表达式](C++ 中的 Lambda 表达式 | Microsoft Docs)来创建线程
  4. 通过带参的方式创建线程：需要用std::ref去包装以传递引用，用于共享变量。可以传递直接传递值。std::thread t1(functionName,std::ref(var));
  5. 通过以智能指针为参数的线程处理函数创建线程。unique_ptr<T>这样可以传递同个对象。方法类同于用带参的方式创建.
  6. 同类的成员函数去创建线程std::thread t1(classname::func,[&/std:ref()]obj)对于传递的对象，可以使用引用也可非引用（此时会调用类的拷贝构造函数生成一个匿名对象）。

线程锁

在多线程操作中，对于非原子操作，若无锁则会导致线程共享的数据不符合预期，为此需要进行加锁。对于所有线程共享锁对象。

锁的创建和操作

使用std::mutex来声明并定义一个锁对象。需要引入头文件#include<mutex>

锁具有lock()和unlock()成员方法，当锁已经被lock时，调用lock()会使线程阻塞直至锁unlock。

此外锁还有另一个操作try_lock()尝试上锁，若上锁不成功，则也不会发生阻塞。同时当上锁成功时，它会返回true。

解决

std::lock_guard

使用std::lock_guard<T>模板,其中T一般为std::mutex。

有两种构造方式：

第一种：std::lock_guard<std::mutex> name(mutex_obj)，在这种模式下，当lock_guard构造时会自动调用mutex_obj的lock()方法以实现上锁。而当lock_guard对象析构时，会自动unlock。

第二种：std::lock_guard<std::mutex> name(mutex_obj,std::adopt_lock)，传入的第二个参数std::adopt_lock会让lock_guard在构造时不调用mutex_obj的lock方法，其含义是假定已经上好锁了。所以在此之前应当上锁。

std::unique_lock

其功能几乎相似于lock_guard，但有些许不同，支持更多的成员函数，比如lock()和unlock()并且可以和condition_variable一起使用控制线程同步。

也有两种构造方法，类似于lock_guard。同样的提供了第二个参数

  1. std::adopt_lock ：认为已经被lock了，构造时不再lock
  2. std::try_to_lock ：尝试上锁，可通过owns_lock()方法判断是否拿到锁。
  3. std::defer_lock ：加上defer_lock用于初始化一个没有加锁的mutex便于后续的成员函数使用

成员函数

  1. lock()：不用自己解锁也可以
  2. unlock()：解锁
  3. try_lock()
  4. release()：释放对mutex的管理权，并返回mutex对象的指针。

此外 unique_lock 还可以使用std::move()实现所有权转交

std::lock() 函数模板

std::lock(mutex1,mutex2……)：一次锁定多个互斥量（一般这种情况很少），用于处理多个互斥量。
如果有一个没锁住，就会把已经锁住的释放掉，然后它就等待，等所有互斥量都可以同时锁住，才继续执行。（要么互斥量都锁住，要么都没锁住，防止死锁）

内存中一个字需要2个地址连续的内存单元来存放，字的低位字节存放在低地址单元中，高位字节存放在高地址单元中
